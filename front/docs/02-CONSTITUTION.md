# 헌법 (Constitution)

> 프로젝트의 상위 구조 원칙. 추상적이고 변하지 않는 방향을 정한다.

---

## 1. 백엔드 없는 프론트엔드 완결 구조

- 이 프로젝트에 백엔드는 없다. 모든 로직은 브라우저에서 실행된다.
- 마크다운 파싱, TTS, 영상 렌더링, 인코딩 모두 클라이언트에서 처리한다.
- 비즈니스 로직은 `lib/`, `hooks/`에서 담당하고, UI 컴포넌트와 분리한다.

## 2. 컴포넌트 경계 분리

- 페이지(Page)와 컴포넌트(Component)는 명확히 분리한다.
- 페이지는 조합만 하고, 컴포넌트는 재사용 가능한 단위로 존재한다.

## 3. 도메인 간 직접 의존 대부분의 경우 금지

- 서로 다른 도메인(기능 영역)의 컴포넌트·훅·유틸은 직접 import하지 않는다.
- 도메인 간 소통이 필요하면 공유 인터페이스(타입, 이벤트)를 통한다.

## 4. 상태는 최소 범위에서 관리

- 전역 상태를 남용하지 않는다.
- 상태는 필요한 가장 가까운 곳에서 선언하고 관리한다.

## 5. 스타일은 Tailwind CSS로 통일

- 별도의 CSS 파일, CSS-in-JS, inline style을 혼용하지 않는다.
- Tailwind CSS를 유일한 스타일링 방식으로 사용한다.

## 6. 타입 안전성 보장

- TypeScript를 사용하며, `any` 타입 사용을 금지한다.
- 모든 props, 응답, 상태에 명시적 타입을 정의한다.

## 7. 점진적 구현 원칙

- 모든 구현은 **실행 가능 지점(Working State)**을 기준으로 한다. 한 번에 많은 것을 만들지 않고, 매 단계마다 동작하는 상태를 유지한다.
- **쉬운 것부터 구현**한다. 단순한 기능을 먼저 완성하고, 복잡한 기능은 그 위에 쌓는다.
